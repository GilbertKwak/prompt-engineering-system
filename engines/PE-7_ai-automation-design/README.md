# 🤖 PE-7 · AI 자동화 설계 및 구현 — 심층 프롬프트 완전판 v1.0

> **역할**: AI 자동화 아키텍트 + 업무 프로세스 컨설턴트 + 시스템 엔지니어  
> **목적**: 사용자의 업무를 분석하고, 최적의 AI 자동화 전략을 설계하며, 실제 구현 가능한 수준까지 구체화  
> **연계**: PE-1 자동개선 · PE-2 자동증식 · PE-3 자동검증

---

## 개요

PE-7은 **AI 자동화 설계 전용 심층 프롬프트 엔진**입니다.  
PE-1~PE-3의 3-Engine(자동개선·자동증식·자동검증)을 자동화 설계 도메인에 특화 적용한 확장판으로,  
추상적 컨설팅이 아닌 **엔지니어가 바로 개발할 수 있는 수준의 아키텍처**를 출력합니다.

---

## 7단계 수행 구조

| 단계 | 명칭 | 핵심 산출물 |
|------|------|------------|
| 1단계 | 업무 구조 분석 | 프로세스 분해 · 병목 식별 |
| 2단계 | 자동화 가능성 평가 | 난이도 매트릭스 (Low/Med/High) |
| 3단계 | AI 적용 설계 | LLM·CV·Prediction 유형 결정 |
| 4단계 | 시스템 아키텍처 설계 | 워크플로우 · 데이터 흐름도 |
| 5단계 | 구현 전략 | MVP → Scaling → Fallback |
| 6단계 | KPI 설계 | 시간절감률 · 오류감소율 · 커버리지 |
| 7단계 | 리스크 및 대응 | 데이터품질 · AI오판 · 보안 |

---

## 3-Engine 연계

| 엔진 | 역할 |
|------|------|
| **PE-1 자동개선** | 프롬프트 초안 품질 자가 진단 → 재작성 루프 적용 |
| **PE-2 자동증식** | 도메인·난이도·포맷별 변형 버전 다중 생성 |
| **PE-3 자동검증** | 5차원 품질 채점 → 합격/재처리 판정 |

---

## 사용 방법

1. `prompt_template.md`를 열어 **입력 정보 섹션**을 작성
2. 작성된 프롬프트를 LLM에 입력
3. 출력된 7개 섹션을 검토 및 피드백 반영
4. PE-1 → PE-2 → PE-3 순서로 3-Engine 후처리 적용
