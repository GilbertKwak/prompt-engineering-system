# 🔁 PE-1: 자동개선 엔진 (Auto-Refinement Engine)

> **Parent**: [prompt-engineering-system](https://github.com/GilbertKwak/prompt-engineering-system)  
> **버전**: v1.1 | **최종 수정**: 2026-04-05

---

## 개요

프롬프트의 **약점을 자동 탐지**하고, 구조적으로 **재작성**하는 엔진입니다.  
최대 3회 반복 루프를 통해 품질을 단계적으로 향상합니다.

---

## 작동 원리

```
Step 1: [진단] 프롬프트 약점 7개 항목 자동 스캔
Step 2: [분류] 결함 유형 분류 (모호성/불완전성/구조불량/범위초과 등)
Step 3: [재작성] 분류된 결함에 따른 타겟 수정
Step 4: [비교] Before/After 품질 지표 비교
Step 5: [반복] 품질 기준 미달 시 최대 3회 루프
```

---

## 진단 체크리스트 (7개 항목)

| # | 항목 | 진단 기준 |
|---|------|----------|
| 1 | 목적 명확성 | 최종 산출물이 1문장으로 정의되어 있는가? |
| 2 | 역할 지정 | AI 역할(페르소나)이 명시되어 있는가? |
| 3 | 컨텍스트 충분성 | 배경 정보가 충분한가? |
| 4 | 제약 조건 | 포맷/길이/언어 제약이 설정되어 있는가? |
| 5 | 예시 포함 | Few-shot 예시가 있는가? |
| 6 | 검증 기준 | 성공 기준이 정량적으로 정의되어 있는가? |
| 7 | 단계적 지시 | 복잡한 태스크가 서브태스크로 분해되어 있는가? |
