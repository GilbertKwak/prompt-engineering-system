# 📋 RCA-001 — 프롬프트 구조 복잡도

> **발행일**: 2026-04-06 | **상태**: ✅ 해결 완료 | **해결 버전**: v3.0
> **Notion**: [RCA-001 Notion Page](https://www.notion.so/33d55ed436f081c593ede0347d2b581a)

---

## 증상 (Symptom)

- XML 중첩 태그 5단 이상 사용 시 파서 실패 매우 잔다 자주 발생
- 에이전트 입출력 형식 미정의 → 결과 변동성 큼
- COT(Chain-of-Thought) 노온 기준 부재 → 구조적 불일치

## 5-Why 분석

| 단계 | Why | 답변 |
|:---:|-----|------|
| 1 | 에이전트 실행 모호? | XML 태그 파싱 실패 |
| 2 | XML 파싱 실패 이유? | 중첩 5단 이상 구조 |
| 3 | 5단 중첩 원인? | 비증적 구조 확장 |
| 4 | 비증적 확장 원인? | 구조 가이드라인 부재 |
| 5 | 가이드라인 부재 이유? | **파싱 안정성 기준 미정의** |

## 근본 원인

> 프롬프트 구조에 파싱 안정성 기준이 없어 XML 중첩 효과로 에이전트 실행 실패 반복

## CAPA

### ✅ CA (Corrective Action — 즉각 시정)

- Markdown 헤더 `##` 기반 구조로 전습 실행
- 에이전트 입출력 형식 명시 (`INPUT:` / `OUTPUT:` 표준화)

### 🛡️ PA (Preventive Action — 재발 방지)

- COT 내부수행 원칙 표준화 (사용자를향한 COT 노출 금지)
- 프롬프트 구조 컨뱌리더 4주 1회 주기 검토

|------|------|------|------|

| 구조적 일관성 | 기준 | +40%p | **향상** |
