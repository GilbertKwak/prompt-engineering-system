## 🔍 근본 원인 분석 (RCA — Root Cause Analysis)

### 문제 1: 프롬프트 복잡도 과다 → 실행 불가

**증상**:
- XML 태그 중첩 과다로 LLM이 구조를 잘못 파싱
- Phase/Agent 구분이 불명확해 응답 시 역할 혼동 발생
- 단일 프롬프트로 실행 시 중간 단계에서 응답 중단

**근본 원인**:
- 시스템 설계 시 "완전성"에 치중, "실행 가능성" 검증 부재
- 에이전트 간 입출력 인터페이스 미정의 (암묵적 의존성)
- COT 지시와 최종 출력 형식 간 충돌 (COT를 노출할지 여부 불명확)

**해결 방안 (v3.0 적용)**:
- XML 구조 → Markdown 헤더 기반 구조로 전환
- 에이전트별 역할·입력·출력을 명시적으로 분리 정의
- "COT는 내부 수행, 결과만 요약 노출" 원칙 명문화
- ORCHESTRATOR를 별도 역할로 분리, 최종 보고서 형식 고정

**검증 방법**:
- 동일 프롬프트를 3회 실행 → 구조적 일관성 확인
- 헬스케어, 금융, 제조 3개 도메인에 적용 → 동일 구조 출력 확인

---

### 문제 2: 검증 기준 모호 → 결과 재현 불가

**증상**:
- 같은 입력에 대해 매번 다른 기회 유형, 다른 스코어 출력
- "시장이 크다"는 주관적 판단, 정량 기준 없음
- Validator가 "낮음/중간/높음"만 출력하고 근거 미제공

**근본 원인**:
- 스코어링 기준이 서술형으로만 존재 (정량 임계값 없음)
- 검증 축의 판단 기준이 에이전트마다 달리 해석됨
- "좋은 기회"의 정의가 없어 SCOUT가 매번 다른 기준으로 선택

**해결 방안 (v3.1 적용)**:
- 스코어링 표에 점수 기준 명시 (TAM $500M+ → 100점 등)
- 검증 6축에 "판단 기준" 열 추가 (정량 기준 병기)
- "통과 기준" 숫자 명시 (총점 70점 이상 → 통과)
- RAG 검색 트리거 쿼리 예시를 각 단계에 포함

**검증 방법**:
- 동일 기회에 대해 스코어링 3회 실행 → ±5점 이내 일관성 확인

### 문제 3: 필수 고려 사항 누락 → 실행 위험

**증상**:
- 출력 보고서에 규제 리스크 언급 없이 "기회"만 강조
- 데이터/프라이버시 위험 미검토로 의료 AI 분야 적용 시 문제 발생
- 플랫폼 리스크 (빅테크 복제 가능성) 누락

**근본 원인**:
- RISK 에이전트가 "시장·경쟁·실행" 위주로 설계, 규제·데이터 리스크 체계적 누락
- 헬스케어·금융 등 규제 집중 도메인에 대한 특수 처리 없음

**해결 방안 (v3.1 적용)**:
- 섹션 0-4 "반드시 고려해야 할 공통 사항" 7개 항목 명문화
- RISK 에이전트에 "규제·법률 리스크" 축 독립 추가
- VALIDATOR에 "법률 자문 필요 여부" 판단 항목 추가
- 헬스케어·금융·공공 도메인에서 규제 리스크를 항상 "최고 우선순위"로 배치

---

### 문제 4: 불필요한 중복 항목 → 응답 길이 과다

**증상**:
- 동일 내용이 ARCHITECT와 VALIDATOR에서 중복 출력
- Executive Summary와 각 섹션 내용이 거의 동일
- XML 내부 설명 텍스트가 실제 출력에 그대로 노출

**근본 원인**:
- 각 에이전트의 출력 범위(scope)가 명확히 정의되지 않음
- "상세 설계"와 "검증"의 경계가 불분명

**해결 방안 (v3.1 적용)**:
- ARCHITECT: 설계에만 집중, 검증은 하지 않음
- VALIDATOR: 검증과 가정에만 집중, 재설계는 하지 않음
- Executive Summary: 3~5문장으로 엄격 제한
- 불필요한 중간 단계 설명 제거, 결과만 출력
