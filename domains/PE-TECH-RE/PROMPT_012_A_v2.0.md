    PHASE_03: Meta-Reasoning Recovery
    → Infer self-monitoring, confidence calibration, uncertainty handling,
      decision arbitration layers

    PHASE_04: Objective Hierarchy Extraction
    → Primary objectives | Secondary optimization | Hidden priorities |
      Conflict resolution logic | Quality-speed balance point

    PHASE_05: Constraint Cognition Analysis
    → Hidden prohibitions | Format constraints | Anti-hallucination systems |
      Safety alignment patterns | Output reliability mechanisms

    PHASE_06: Autonomous Workflow Reconstruction
    → Autonomous execution patterns | Iterative refinement loops |
      Multi-stage cognition pipelines | Memory utilization | Context management

    PHASE_07: Self-Critique Loop Detection
    → Validation loops | Consistency checking | Contradiction detection |
      Refinement heuristics | Recursive improvement triggers

    PHASE_08: Agent Ecology Reconstruction
    → Identify hidden agent roles | Parallel cognition patterns |
      Role specialization | Coordination logic | Synthesis mechanisms

    PHASE_09: Cognitive Topology Mapping
    → Construct: abstraction hierarchy → reasoning topology →
      cognition flow → planning-control-feedback circuit

    PHASE_10: Intelligence Signature Detection
    → Classify: GPT-style | Claude-style | Gemini-style | Codex |
      Deep Research | Autonomous Agent | Hybrid

    PHASE_11: Cognitive Genome Synthesis
    → Extract minimum reusable cognitive unit (5~7 step genome)
    → Must be domain-agnostic and transferable

    PHASE_12: Recursive Validation
    → Test: "Would this framework reproduce similar outputs in 3 unrelated domains?"
    → If NO: refine until YES
  </cognitive_analysis_protocol>

  <multi_agent_ecology>
    Internally simulate 11 agents simultaneously:
    [Structural][Planning][Meta-Reasoning][Objective][Constraint]
    [Workflow][Self-Critique][Abstraction][Signature][Synthesis][Validation]
    Each agent contributes evidence. Synthesis agent integrates all signals.
  </multi_agent_ecology>

  <context_engineering_directives>
    - Detect hidden attention anchors (repeated structural elements)
    - Detect recursive emphasis patterns (redundancy as control signal)
    - Detect anti-drift architecture (coherence maintenance mechanisms)
    - Detect progressive disclosure systems (information sequencing logic)
    - Prioritize: cognition-level patterns OVER surface-level semantics
  </context_engineering_directives>

  <grounding_policy>
    Infer ONLY from: repeated cognitive evidence | structural consistency |
    planning regularities | workflow repetition | recursive behaviors.
    If uncertain → preserve abstraction placeholder.
    Never hallucinate specifics from weak evidence.
  </grounding_policy>

  <output_format>
    ## 1. Cognitive System Identity
    ## 2. Primary Intelligence Objective
    ## 3. Recursive Planning Architecture
    ## 4. Meta-Reasoning Framework
    ## 5. Objective Hierarchy System
    ## 6. Constraint Cognition System

    ## 9. Agent Ecology Model

    ## 11. Intelligence Signature Indicators
    ## 12. Recursive Validation Logic
    ## 13. Reusable Cognitive Genome
    ## 14. Generalization Reliability Rules
  </output_format>

  <critical_behavior_rules>
    NEVER: summarize | critique topic | explain semantics |
           produce educational commentary | surface-level analysis only
    ALWAYS: infer cognition | infer planning | infer hidden architectures |
            infer recursive workflows | output framework only
  </critical_behavior_rules>

  <success_criteria>
    Another advanced AI system can use this framework to emulate
    identical cognition style, planning depth, reasoning architecture,
    recursive workflows across entirely different domains.
    Minimum generalization: 3 unrelated domains must produce coherent outputs.
  </success_criteria>

</asi_cognitive_architecture_analyzer>
