## 3. 엔진별 업그레이드 상세

### 3.1 PE-1 자동개선 엔진 (Auto-Refinement)

**업그레이드 포인트**:
1. **Chain-of-Thought 구조 강화** — 단순 약점 나열 → 원인-영향-수정 3단계 구조화
2. **약점 탐지 정밀도 향상** — 7개 도메인별 진단 기준 카탈로그 기반
3. **루프 종료 조건 개선** — 3회 고정 → 품질 점수 85+ 도달 시 조기 종료

**성과** |
- 평균 루프 횟수: 2.8회 → **1.9회** (-32.1%)
- 개선 후 평균 품질 점수: 68 → **87** (+27.9%)

### 3.2 PE-2 자동증식 엔진 (Auto-Proliferation)

**업그레이드 포인트**:
1. **5축 → 8축 변형 매트릭스 확장** — 기존 도메인·난이도·포맷에 언어(KR/EN)·톤(Formal/Casual) 추가
2. **도메인 크로스오버 변형** — 단일 도메인 험시탈을 벗어나 인접 도메인 하이브리드 생성
3. **븴드리업 평가 연동** — 변형 뺄드 시 PE-3로 자동 전달되도록 파이프라인 연결

**성과**:
- 변형 버전 수: 5종 → **8종** (+60.0%)
- 도메인 커버리지: 3개 → **7개** (+133%)

### 3.3 PE-3 자동검증 엔진 (Auto-Validation)

**업그레이드 포인트**:
1. **5차원 스코어링 가중치 재조정** — 명확성(25%) · 완전성(25%) · 실행가능성(20%) · 안전성(15%) · 효율성(15%)
2. **합격선 상향**: 70점 → **75점** (보수적 품질 기준 강화)
3. **신뢰도 가중치 시스템** — 각 차원별 평가자 역할 섹션 신뢰도 반영
4. **재처리 피드백 루프** — 불합격 판정 시 PE-1에 특정 개선 지시를 포함하여 자동 반환

**성과**:
- 1차 합격률: 61% → **84%** (+37.7%)
- 평균 검증 소요 시간: 8.2분 → **3.1분** (-62.2%)

## 5. 보고서 작성 체계 수립 (v1.5에서 신규 첨가)

### 5.1 작성 순서: **한글 先 → 영문 後** 확정

4가지 근거로 **한글 먼저 작성** 순서를 확정했습니다:

1. **사고 흐름 자연성** — 모국어(KR)로 논리 구조 먼저 확립 후 영문 재표현
2. **LLM 프롬프트 효율** — 한글 초안 제공 시 hallucination 위험 감소
3. **검증 이중화** — 한글(논리·사실) + 영문(용어·표현) 검증 레이어 분리
4. **수정 비용 최소화** — 한글 단계 수정 시 영문 재작업 범위 유한

### 5.2 12단계 문제 사전방지 체계 (판단위험 구조)

보고서 오류의 **78%는 구조 설계·자료 수집 단계**에서 발생합니다.

| 단계 | 작업 | 위험 등급 |
|------|------|----------|
| 01 목적·범위 정의 | KR+EN 범위 동시 확정 | 🔴 Critical |
| 02 목차 설계 | 논리 흐름 검증 | 🔴 Critical |
| 03 자료 수집 | 출처 메타데이터 즉시 기록 | 🔴 Critical |
| 04 자료 검증 | PE-3 자동검증 적용 | 🔴 Critical |
| 05 KR 초안 작성 | PE-1 자동개선 적용 | 🟠 High |
| 06 KR 1차 검토 | PE-3 5차원 채점 | 🟠 High |
| 07 KR Final 확정 | GitHub 버전 태깅 | 🟡 Medium |
| 08 EN 작성 | PE-2 자동증식 적용 | 🟠 High |
