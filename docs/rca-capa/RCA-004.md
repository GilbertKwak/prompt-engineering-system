## 증상 (Symptom)

- 에이전트별 동일 콘텐츠 중복 생성
- Executive Summary가 무제한 확장되어 중복 요약 다수 포함
- 승인 프로세스 지연 원인

## 5-Why 분석

| 단계 | Why | 답변 |
|:---:|-----|------|
| 1 | 중복 콘텐츠 원인? | 에이전트 출력 범위 미정의 |
| 2 | 범위 미정의 원인? | 에이전트간 역할 분리 부재 |
| 3 | 역할 분리 부재 원인? | 엔지니어링 단계에 서로 무시 |
| 4 | 서로 무시한 원인? | 쫐독 에이전트 부재 |
| 5 | 쫐독 에이전트 부재 원인? | **시스템 설계에 Orchestrator 역할 정의 누락** |

## 근본 원인

> 시스템 설계 시 orchestrator 에이전트의 역할 및 scope 미정의

## CAPA

### ✅ CA (Corrective Action)

- 에이전트별 역할 경계 명확화 (RACI 매트릭스)
- Executive Summary 3~5문장 제한 적용

### 🛡️ PA (Preventive Action)

- 신규 에이전트 설계 시 scope 맴튤다임 의무화
- 매월 에이전트 역할 검토 럼트라인 도입
