## MAINTENANCE ENGINE

### 4-Loop 자동 유지보수 시스템
```
[Loop 1: AUTO VALIDATE]
  - 주기: 매일 (GitHub Actions 트리거)
  - PE-3 자동검증 전체 스캔
  - 점수 하락 감지 → 알림 발행

[Loop 2: AUTO IMPROVE]
  - 트리거: PE-3 점수 < 85 감지 시
  - PE-1 자동개선 엔진 투입
  - 최대 3회 재작성 루프
  - 개선 결과 → GitHub PR 자동 생성

[Loop 3: AUTO SYNC]
  - 주기: GitHub 커밋 이벤트 감지 시
  - Notion ↔ GitHub 불일치 탐지
  - 자동 동기화 실행 (승인 모드 선택 가능)

[Loop 4: AUTO AUDIT]
  - 주기: 월 1회
  - 전체 포트폴리오 감사
  - 갭 분석 (OPT-PA-04) 자동 실행
  - 감사 보고서 생성 → Notion 자동 업데이트
```

### 오류 탐지 규칙
| 오류 유형 | 탐지 방법 | 자동 대응 |
|---|---|---|
| 깨진 링크 | URL 유효성 검사 | 오류 플래그 + 수정 제안 |
| 버전 불일치 | Notion vs GitHub 비교 | 자동 동기화 PR |
| 품질 하락 | PE-3 재채점 | PE-1 개선 루프 투입 |
| 미사용 자산 | 사용 이력 분석 (90일) | Archive 후보 알림 |
| 중복 자산 | 유사도 분석 (>85%) | 통합 검토 권고 |
