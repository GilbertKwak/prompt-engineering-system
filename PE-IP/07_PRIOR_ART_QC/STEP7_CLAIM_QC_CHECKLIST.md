## Q-01 : 청구항 1 독립항 — 시스템 청구항

**현행 문언 (핵심)**:
> "HBM-PIM 장치와 DDR5 장치를 포함하는 이종 메모리 시스템에서 동작하는 SW 오케스트레이터로서, (a) 연산 그래프의 각 노드에 대해 실행 시점에 동적으로 PA·BS·CR·AF 속성을 결정하는 TSAT 모듈; (b) 상기 속성 벡터 V_t를 입력으로 규칙 기반 정책(RBP)과 학습 기반 정책(LBP)의 앙상블을 통해 배치 장치를 결정하는 HMPDE 모듈; (c) 결정된 배치에 따라 aimx_malloc 또는 numa_alloc을 호출하는 메모리 할당자 모듈; (d) 복수의 트리거 조건에 따라 HBM-PIM과 DDR5 간 데이터 마이그레이션을 수행하는 RMS 모듈; (e) PIM 연산 장치 활용률을 수집하여 상기 TSAT 및 HMPDE의 파라미터를 자동 갱신하는 PUFL 모듈;"

### QC 항목

| 항목 | 검토 내용 | 판정 | 비고 |
|------|----------|------|------|
| 1-A | 전제부: 이종 메모리 시스템 명시 | ✅ | P-08(NVIDIA) 대비 차별점 확보 |
| 1-B | (a) "실행 시점에 동적으로" 문언 | ✅ | N-01(HeteroSched) 오프라인과 차별 |
| 1-C | (b) RBP+LBP 앙상블 명시 | ✅ | 구조 명확 |
| 1-D | (c) aimx_malloc / numa_alloc 구체적 호출 명시 | ✅ | 실시예와 정합 |
| 1-E | (d) "복수의 트리거 조건" — 조건 수 불특정 | ⚠️ | "M1·M2·M3 중 적어도 하나"로 보강 권고 |
| 1-F | (e) PUFL: 파라미터 갱신 대상 명시 | ✅ | τ_PA 및 LBP 가중치 포함 확인 |
| 1-G | 청구항 말미 "이를 특징으로 하는 소프트웨어 오케스트레이터" 기재 | ✅ | 적합 |

**종합 판정**: ⚠️ 1-E 항목 보강 후 통과

**보강 문언안**:
```
(d) HBM-PIM 장치의 메모리 점유율이 상한 임계값(θ_hbm)을 초과하는 제1 트리거(M1),
    PIM 연산 장치의 활용률(TUR)이 하한 임계값(TUR_low) 미만인 제2 트리거(M2), 및
    워크로드 유형이 전환되는 제3 트리거(M3) 중 적어도 하나의 트리거 조건 충족 시
    HBM-PIM 장치와 DDR5 장치 간 데이터 마이그레이션을 수행하는 RMS 모듈;
```

## Q-04 : 청구항 4 종속항 — 앙상블 가중치 α

**현행 문언**:
> "제1항에 있어서, HMPDE 모듈은 α·RBP(V_t) + (1-α)·LBP(V_t) 앙상블 함수를 사용하는 것을 특징으로 하는 소프트웨어 오케스트레이터"

### QC 항목

| 항목 | 검토 내용 | 판정 | 비고 |
|------|----------|------|------|
| 4-A | P-07(SK Hynix) 대비: α 동적 갱신 명시 | 🔴 | **필수 보강** — 현행 문언에 α 동적 갱신 미기재 |
| 4-B | α 값 범위 명시 | ⚠️ | [0, 1] 범위 기재 권고 |
| 4-C | PUFL 연동 명시 | 🔴 | **필수 보강** — 피드백 루프 연결 근거 |

**종합 판정**: 🔴 재작성 필요 (4-A, 4-C 필수)

**보강 문언안**:
```
제1항에 있어서, 상기 HMPDE 모듈은,
  P(V_t) = α·RBP(V_t) + (1-α)·LBP(V_t)  (단, α∈[0,1])
로 정의되는 앙상블 함수를 사용하되,
상기 가중치 α는 상기 PUFL 모듈로부터 수신한 HBM-PIM 활용률 이력(TUR_history) 및
추론 지연 피드백(latency_delta)에 기반하여 런타임에 동적으로 갱신되는 것을
특징으로 하는 소프트웨어 오케스트레이터.
```

## Q-05 : 청구항 5 종속항 — RMS 트리거 조건

**현행 문언**:
> "제1항에 있어서, RMS 모듈은 HBM 점유율 초과, PIM 활용률 저하, 워크로드 전환의 세 가지 조건에 따라 마이그레이션을 수행하는 것을 특징으로 하는 소프트웨어 오케스트레이터"

### QC 항목

| 항목 | 검토 내용 | 판정 | 비고 |
|------|----------|------|------|
| 5-A | P-03(KAIST) 대비: M2 조건(TUR_low) 명시 | 🔴 | **필수 보강** — KAIST는 M1만 개시 |
| 5-B | 임계값 파라미터(θ_hbm, TUR_low) 기재 | ⚠️ | 설정 가능한 값임을 명시 권고 |
| 5-C | M3(워크로드 전환) 판별 방법 명시 | ⚠️ | "연산 그래프 노드 유형 전환 감지" 추가 권고 |

**종합 판정**: 🔴 5-A 필수 보강

**보강 문언안**:
```
제1항에 있어서, 상기 RMS 모듈은,
  (a) HBM-PIM 장치의 메모리 점유율이 설정 가능한 상한 임계값(θ_hbm)을 초과하는 제1 트리거(M1);
  (b) PIM 연산 장치의 활용률(TUR)이 설정 가능한 하한 임계값(TUR_low) 미만으로 하락하는 제2 트리거(M2); 및
  (c) 연산 그래프 내 노드의 주요 유형이 PIM 적합 유형에서 비적합 유형으로 전환되는 제3 트리거(M3)
중 적어도 하나의 조건이 충족될 때 HBM-PIM 장치와 DDR5 장치 간 데이터 마이그레이션을
비동기적으로 수행하는 것을 특징으로 하는 소프트웨어 오케스트레이터.
```

## Q-06 : 청구항 6 종속항 — PUFL 피드백 루프

**현행 문언**:
> "제1항에 있어서, PUFL 모듈은 PIM 연산 장치 활용률을 수집하고 임계값과 비교하여 τ_PA를 자동 갱신하는 것을 특징으로 하는 소프트웨어 오케스트레이터"

### QC 항목

| 항목 | 검토 내용 | 판정 | 비고 |
|------|----------|------|------|
| 6-A | N-04(arXiv PUFL) 대비: 이종 메모리 비율 기재 | 🔴 | **필수 보강** — 논문은 단일 메모리 타입 |
| 6-B | TUR_high / TUR_low 양방향 임계값 명시 | ⚠️ | 단방향으로 기재된 경우 양방향 보강 |
| 6-C | LBP 재학습 데이터 공급 경로 명시 | ⚠️ | "LBP 모델 파라미터 갱신" 추가 권고 |

**종합 판정**: 🔴 6-A 필수 보강

**보강 문언안**:
```
제1항에 있어서, 상기 PUFL 모듈은,
  HBM-PIM 장치의 활용률(PPUUR)과 DDR5 장치의 활용률의 비율을 수집하고,
  상기 비율이 상한 임계값(TUR_high)을 초과하는 경우 τ_PA를 상향 조정하고,
  상기 비율이 하한 임계값(TUR_low) 미만인 경우 τ_PA를 하향 조정하며,
  상기 조정 이력을 상기 HMPDE 모듈의 LBP 모델 파라미터 재학습 데이터로 공급하는 것을
특징으로 하는 소프트웨어 오케스트레이터.
```
