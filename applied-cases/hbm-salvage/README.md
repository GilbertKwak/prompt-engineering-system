## 📋 작성 규칙 (Writing Protocol) — Rule 0 ~ Rule 3

> 모든 HBM Salvage 보고서 작성 시 아래 4개 규칙을 **의무 적용**합니다.

| Rule | 이름 | 내용 |
|------|------|------|
| **Rule 0** | 사전 확인 (Pre-Check) | 보고서 작성 전 반드시 **Notion 허브 & GitHub 리포지토리를 조회**하여 기존 진행 상황·완료 챕터·미완료 항목을 확인한다 |
| **Rule 1** | 병렬 작성 (Bilingual) | 모든 챕터는 **KO·EN 병렬 구조**로 작성하며, PE-3 자동검증 체크리스트를 포함한다 |
| **Rule 2** | 사후 업데이트 (Post-Update) | 보고서 작성 완료 후 **Notion 페이지와 GitHub 리포지토리를 동시 업데이트**한다 |
| **Rule 3** | 목록 출력 (Status Report) | 매 세션 종료 시 **전체·완료·미완료 목록**을 표로 출력한다 |
