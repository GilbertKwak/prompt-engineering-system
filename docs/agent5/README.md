# 🤖 PE-5 마스터 오케스트레이터 (Master Orchestrator)

> **에이전트 ID**: PE-5 | **버전**: v0.3 | **상태**: 🟡 개발 중 | **업데이트**: 2026-04-09

[![Master Index](https://img.shields.io/badge/←%20Back-Master%20Index-gray)](../index.md)

---

## 📌 목적 (Purpose)

PE-1~4 에이전트를 조율하고, 전체 파이프라인의 실행 흐름을 관리합니다. 중복 출력 방지 및 역할 경계 유지가 핵심 책임입니다.

---

## ⚙️ 에이전트 역할 카드

| 항목 | 내용 |
|------|------|
| **역할** | PE-1~4 에이전트 조율 및 파이프라인 실행 관리 |
| **입력 (Input)** | 사용자 요청, 에이전트별 출력 결과 |
| **출력 (Output)** | Executive Summary (3~5문장), 최종 통합 결과 |
| **의존성** | PE-1, PE-2, PE-3, PE-4 전체 |
| **핵심 제약** | Executive Summary ≤ 5문장, 중복 출력 금지 |
| **라우팅 로직** | 스코어 < 70 → PE-1 재개선 / ≥ 70 → PE-2 증식 또는 PE-4 적용 |

---

## 🔄 오케스트레이션 흐름

```
사용자 요청
    ↓
PE-5 (수신 및 라우팅)
    ├─→ PE-1 (개선) → PE-3 (검증)
    │       ↑ 실패 시 재순환
    ├─→ PE-2 (증식) ← PE-3 통과 후
    └─→ PE-4 (적용) ← 최종 승인 후
         ↓
    Executive Summary 출력
```
